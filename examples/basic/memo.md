# fivepixels 랜딩·사이트 기획 메모

> 이 문서는 랜딩페이지·가이드 사이트를 만들 때 참고하는 **기획 원본**입니다.  
> Codex·에이전트가 읽고 페이지를 구현할 때 **비개발자 설득**과 **개발자 안심**을 동시에 만족하도록 구성했습니다.

---

## 0. 문서 목적 & 설득 균형

| 독자 | 랜딩에서 원하는 반응 | 문서 내 위치 |
|------|---------------------|-------------|
| 비개발자 (PM·기획·프리랜서·클라이언트) | “우리 상황에 맞다, 가볍게 시작할 수 있겠다” | §1~§4, §6, `/for`, `/guides` |
| 개발자 | “도입 비용 낮고, 기존 흐름 안 깨고, 점진 적용 가능” | **§5 개발자 안심 블록**, `/docs` |
| 둘 다 | 홈에서 각자 필요한 CTA로 갈라짐 | 히어로 이중 CTA, §7 사이트맵 |

**홈(`/`) 최소 기준 (협상 불가):**

| 독자 | 홈에서 반드시 받아야 하는 것 |
|------|------------------------------|
| 비개발자 | **공감** — “캡처·슬랙·저기요” Pain이 바로 이해됨 |
| 개발자 | **도입 리스크 해소** — 비용·점진 적용·기술 근거가 **추상 말이 아닌 사실**로 보임 |

> “100% 같은 강도로 동시 설득”은 어렵다는 말은 **디자인 밀도** 이야기일 뿐, 위 최소 기준은 **둘 다 홈에서** 충족해야 한다. 개발자 설득을 `/docs`로만 미루면 안 됨.

**구조 vs 구현 리스크:** 이 memo의 **문서 설계**는 위 기준에 가깝다. 남은 리스크는 **실제 랜딩 카피·디자인**에서 추상 마케팅 문구로 흐려질 때 — §13·§14로 방지.

## 1. 포지셔닝

### 한 줄 (비개발자 히어로)

> **무거운 QA 인프라 없이, 스테이징 화면 위에서 피드백을 모으고 끝까지 따라가세요.**

### 한 줄 (개발자 서브 / §5 블록 헤드라인)

> **기존 React 앱 위에 얇게 붙이고, 1페이지부터 점진적으로 확장하세요.**

### 대체하는 것 / 아닌 것

| 대체 (Pain) | 아닌 것 |
|-------------|---------|
| 캡처 + 슬랙/카톡 + “저기요” | 엔터프라이즈 QA 플랫폼 |
| 노션·메일에 흩어진 수정 요청 | 별도 로그인하는 피드백 SaaS |
| 무거운 도입 프로젝트만 하다 미루기 | 브라우저 자동화·E2E 테스트 대체 |

### 금지 (법적·포지셔닝)

- **경쟁사·타 제품명 직접 비교** (특정 QA SaaS, 자동화 툴 이름)
- 비교는 **「지금 방식 vs fivepixels」** 만 (캡처·메신저·구두)

| 2 | Pain (4카드) | “몇 번째 버튼?” / 캡처·메신저 산재 / QA 인프라 부담 / 클라이언트 피드백 모호 |

### 히어로 카피 (확정안 — 구현 시 그대로 쓰거나 미세 수정만)

| 요소 | 확정 카피 | 피할 표현 |
|------|----------|----------|
| 헤드라인 | **스크린샷과 슬랙 대신, 검수하는 그 화면에서 바로 피드백하세요** | “혁신적인 QA 솔루션”, “올인원 플랫폼” |
| 서브 | 무거운 QA 툴 없이도 됩니다. 클릭한 위치에 남기고, 팀과 확인하고, 필요할 때만 이슈로 넘깁니다. | “DX 극대화”, “워크플로우 혁신” |
| 개발자 한 줄 (히어로 하단, 작은 텍스트) | React 앱에 얇게 붙입니다. 반나절이면 스테이징 1페이지부터 시작할 수 있어요. | Shadow DOM·localStorage 등 **용어 나열** (§5에서 풀어 씀) |

**히어로 자가 점검:** 비개발자가 **5초 안에** “아, 캡처 안 해도 되겠다”가 와야 함. 기술 용어는 히어로 본문에 넣지 않음.

### 히어로 CTA (이중 — 분기 필수)

| 버튼 | 문구 | 링크 | 독자 | 구현 노트 |
|------|------|------|------|----------|
| Primary | 데모에서 2분 만에 써 보기 | `/demo` | 비개발자·의사결정 | 시각적으로 **더 강조** (filled) |
| Secondary | 5분 설치 가이드 | `/docs/setup` (1순위) 또는 `/guides/quickstart#dev` | 개발자 | “개발자용” 라벨 가능, **숨기거나 푸터만 X** |

**CTA 자가 점검:** 스크롤 없이 히어로에서 **두 갈래**가 보이는가. 개발자 CTA가 “문서”가 아니라 **“5분·설치·시작”** 뉘앙스인가.

## 4. 지금 방식 vs fivepixels (홈 §7)

```tsv
지금	fivepixels
캡처 + 화살표	화면 위 마커 (위치 고정)

"저기 버튼이요"	클릭한 요소에 코멘트

```

## 5. 개발자 안심 블록 ★ (랜딩 필수)

> 홈 섹션 순서 **5번** (Pain·해결·UI Edit 직후). `For developers` 라벨 권장.  
> **접힌 블록·푸터 전용·`/docs`로만 대체 금지.**

### 구현 시 눈에 띄게 (디자인 가이드)

| 항목 | 권장 | 피하기 |
|------|------|--------|
| 위치 | 첫 스크롤~2스크롤 이내, UI Edit 다음 **독립 섹션** | 페이지 최하단, FAQ 아래 |
| 시각 | 배경색·보더로 본문과 **구역 분리**, 섹션 제목 `For developers` | Pain 카드와 동일 스타일로 묻힘 |
| 밀도 | 아래 **필수 5키워드**가 본문에 각각 1회 이상 | “가볍다”, “쉽다”만 반복 |
| 코드 | 스니펫 **기본 펼침** 또는 1클릭 확장 (숨김 기본 X) | “자세히는 문서에서”만 |

### 헤드라인

**개발자에게 — 도입은 작게, 유지는 가볍게**

### 필수 안심 근거 5종 (랜딩 본문에 빠지면 안 됨)

구현·카피 QA 시 아래 **키워드/개념**이 §5 안에 **각각 명시**되어 있는지 확인:

```tsv
필수 근거	랜딩에 쓸 구체 문장 (예시)	대응 기술
도입 비용	반나절이면 npm install 후 스테이징 1페이지에 붙일 수 있습니다	npm + <FivePixels /> 1회
점진 적용	전체 앱이 아니라 검수 많은 페이지·요소에만 data-report-id	1페이지부터 확장
Shadow DOM	피드백 UI는 Shadow DOM — 호스트 앱에 CSS import 없음, 스타일 충돌 없음	Shadow Root
localStorage 체험	API 없이도 localStorage로 팀 체험 가능 (handler 생략 시)	기본 adapter
devOnly	visibility.devOnly로 프로덕션 비노출, 스테이징에서만 켜기	visibility
```

> §5에 “섹션이 있다”만으로는 불충분. 위 **5종이 문장으로** 들어가야 함.

### 5가지 안심 메시지 (카드 또는 표)

```tsv
개발자가 궁금한 것	fivepixels 답변	랜딩 한 줄 카피
세팅 비용이 낮은가?	npm install + <FivePixels /> 1회 + data-report-id	반나절이면 스테이징 1페이지부터
점진 도입 가능한가?	한 페이지·한 영역만 id 붙여 시작	전체 앱 한꺼번에 태깅할 필요 없음
기존 워크플로우를 깨나?	피드백 UI는 Shadow DOM, 호스트 CSS import 없음	검수 UI가 앱 스타일을 망가뜨리지 않음
기술적으로 어떻게 붙나?	React 18+ 컴포넌트 1개 마운트, handler 생략 시 localStorage	백엔드 없이도 팀 체험 가능
유지보수 부담은?	devOnly로 prod 비노출, visibility로 on/off	스테이징·내부에서만 켜 두기
```

### 점진 도입 3단계 (랜딩 다이어그램용)

```
Day 0   npm install + <FivePixels /> + visibility.devOnly
Day 1   검수 많은 페이지 1곳에만 data-report-id
Day 2+  handler 연동 / GitHub / 팀 규칙 (/guides/rules)
```

### 왜 SaaS가 아니라 라이브러리인가

```tsv
SaaS 피드백 툴의 부담	fivepixels (라이브러리)
별도 계정·온보딩	이미 쓰는 스테이징 URL 그대로
월 구독·시트 제한	MIT, 자체 호스팅 앱 위에 임베드
외부 서버에 화면 캡처 업로드	피드백 데이터는 localStorage 또는 **내 API**
클라이언트가 또 하나의 툴 학습	**보고 있던 제품 화면**에서 코멘트
```

### 개발자용 한 줄 (§5 푸터)

비개발자는 화면에서 남기고, 개발자는 **기존 이슈·PR 흐름**으로 처리. 필요할 때만 `onEvent` / GitHub 연동.

### 코드 스니펫 (랜딩 접힌 블록 — 최소)

```tsx
// 1) 설치
npm install @fivepixels-js/react react react-dom

// 2) 앱 루트에 한 번
<FivePixels project={{ id: "my-app" }} visibility={{ devOnly: true }} />

// 3) 검수 대상에만
<button data-report-id="hero-cta">시작하기</button>
```

상세: `/docs/setup`

## 8. `/guides` 목차 요약

### `/guides/quickstart`

- **비개발자:** URL → 피드백 추가 → 클릭 → 작성 (단축키 카드 이미지)
- **개발자:** install → `FivePixels` → `data-report-id` 1개 (5분)

### `/guides/rollout` — 1주

```tsv
일차	할 일	성공 기준
Day 1	스테이징 1페이지에만 붙이기	devOnly, id 1~3개
Day 2	내부 2~3명 사용	피드백 5건
Day 3	/rules 합의	1페이지 규칙
Day 4	(프리랜서) 클라이언트 1차 검수	/guides/client 발송
Day 5	미해결만 보기	View 모드 + 필터
```

### `/guides/rules` (복사 템플릿)

```tsv
규칙	내용
어디에	스테이징 URL만 (캡처 메일 지양)
이름	id는 kebab-case (hero-cta)
한 수정 한 케이스	여러 요청은 케이스 분리
해결	스테이징 재확인 후 resolved
클라이언트	안내문 + 클릭으로 표시
```

### `/guides/client` — 프리랜서 발송문 골격

- 제목: 「스테이징에서 직접 표시해 주세요」
- 본문: URL / 클릭 방법 / “저기요” 대신 화면에 남기기
- 스크린샷 3장: 모드 전환 → 클릭 → 작성

### `/guides/workflow`

```
남김(open) → (선택) UI Edit → 확인 요청 → 수정 → 재확인 → 해결(resolved)
→ (성장 단계) GitHub Issue
```

## 11. 부록 — 제품 사실 (랜딩 히어로 X, `/docs`·카피 검증용)

### 제품 한 줄 (기술 정확)

스테이징·QA·내부 도구에서 DOM 요소 단위 피드백을 남기고, 팀과 답변·검수 후 GitHub Issue로 승격할 수 있는 React 라이브러리 (`@fivepixels-js/react`, MIT).

### 랜딩용 핵심 8 (양쪽 타깃 혼합)


## 13. 구현 시 빠지기 쉬운 설득 포인트 (카피 안티패턴)

문서 구조는 유지하고, **실제 화면**에서 아래로 흐려지지 않도록:

```tsv
안티패턴	증상	대신 쓸 것
추상 마케팅만	"가볍고 혁신적인 QA"	§3 히어로 확정안, §4 비교표
개발자 설득을 /docs로만	홈엔 감성 카피만	§5 필수 5종 문장
§5를 접거나 최하단	개발자가 스크롤 안 함	§5 디자인 가이드 (배경·제목·코드 펼침)
기술 용어 히어로 나열	비개발자 이탈	히어로=공감, §5=근거
CTA 하나만	개발자 게이트키퍼 이탈	이중 CTA, Secondary 시각적으로도 존재
"쉽다" 반복	개발자 신뢰 하락	devOnly, localStorage, 반나절, 1페이지 등 **수치·사실**
```

### 카피 Before / After (§5용)

| ❌ 구현 시 흔한 약한 문장 | ✅ memo 권장 문장 |
|--------------------------|------------------|
| 개발도 쉽습니다 | `npm install` 후 `<FivePixels />` 한 번, **반나절**이면 1페이지 |
| 부담 없이 도입 | **1페이지·몇 개 요소**만 `data-report-id` — 전체 태깅 불필요 |
| 스타일 안 깨짐 | **Shadow DOM** — 호스트 CSS import 없음 |
| 서버 없이 OK | handler 생략 시 **localStorage**로 팀 체험 |
| 프로덕션 안전 | **`visibility.devOnly`** — 스테이징에서만 노출 |
